🎮 游戏作品集 + 🎬 仿奈飞影片网站 学习与开发清单

目标：开发两个独立的跨平台项目，覆盖全栈、移动端、AI等全部技能，工程环境兼容 Windows 和 macOS。

一、全局跨平台工具链

工具 / 环境 用途 跨平台安装方式
Node.js (LTS) 前端运行环境 官网下载安装包
VS Code 编辑器 官网下载，登录同步插件
Git 版本控制 安装 Git，配置全局用户信息
Docker Desktop 数据库与向量数据库 官网下载，启动后运行容器
Python 3.10+ 后端/AI脚本 官网安装，确保 python 和 pip 可用
Flutter SDK 移动端开发（可选） 官网下载解压，配置 PATH
Cursor AI 辅助编码 下载对应系统版本
Chrome 浏览器 插件调试、H5测试 官方安装
Axure RP 原型设计 下载对应系统版本（Win/Mac 均有）
Postman API 调试 跨平台客户端

所有工具原生跨平台，代码使用标准 API，路径统一为正斜杠。

二、项目开发路线图

🎮 项目A：游戏作品集展示网站

展示你开发的所有游戏，包含可玩演示/视频、技术说明、截图等，专业个人品牌站。

A1. 静态原型与快速上线

  • 使用 Vite + vanilla-ts 初始化项目，Git 仓库管理
  • 用 Axure 绘制首页、详情页、关于页线框图
  • 使用 Tailwind CSS 构建响应式布局（游戏卡片网格）
  • 嵌入游戏 WebGL 演示（<iframe>）或玩法视频（<video>）
  • 制作游戏截图 gallery（可用 JS 简单灯箱）
  • 部署到 Vercel/Netlify，获得公开链接

A2. React 重构与组件化

  • 用 Vite 创建 React + TypeScript 项目
  • 安装 Ant Design，搭建 Layout、导航、卡片组件
  • 使用 React Router 实现多页面（首页、游戏库、游戏详情、关于我）
  • 封装 GameCard、GameDetail、ImageGallery 等组件
  • 使用 Axios 从本地 JSON 或 Markdown 文件加载游戏数据
  • 移动端适配（响应式断点、触摸滑动、横屏视频处理）

A3. 后端接入与动态内容

  • 用 FastAPI (Python) 或 Spring Boot (Java) 搭建后端
  • 设计 MySQL 数据库 games 表（标题、描述、标签、截图URL、视频URL、技术栈等）
  • 实现简单的 REST API：获取列表、获取详情
  • 可选：后台管理界面（React + Ant Design Pro）用于增删改游戏信息
  • 文件上传：演示包/视频存放到对象存储或本地静态目录
  • 前端接入真实 API，替换模拟数据

A4. 数据可视化与个人特色

  • 使用 ECharts 展示个人技术栈雷达图、游戏类型分布饼图
  • 如果有游戏数据（如试玩次数），可视化展示（否则用静态数据）
  • 用 Axure 制作高保真交互原型，导出 HTML 作为作品集附件

A5. 跨平台扩展（移动端 + 插件）

  • 用 Flutter 实现一个简单 App，展示游戏列表和详情（可选）
  • 用 React Native (Expo) 实现同样功能，对比学习
  • 开发 Chrome 新标签页插件，每次打开随机展示一款游戏

A6. AI 智能加持

  • 用 Python 脚本为所有游戏描述生成 Embedding，存入 Qdrant（Docker）
  • 后端增加语义搜索接口，用户可自然语言搜索（如“像素风格动作游戏”）
  • 前端搜索框接入智能搜索，提升体验

🎬 项目B：仿奈飞自制影片网站

独立视频流媒体平台，展示你自制的影片，包含用户系统、播放器、推荐等，体现全栈能力。

B1. 原型设计与框架搭建

  • 用 Axure 绘制主要页面：登录页、首页（分类横幅+海报墙）、详情播放页、搜索页
  • 使用 Vite + React + TypeScript 初始化项目
  • 采用 Tailwind CSS 或 styled-components 实现 Netflix 风格暗色主题
  • 使用 Ant Design 或自建组件，设计可复用的 MovieRow（横向滚动）、Banner 组件

B2. 核心播放功能

  • 集成 video.js 或 react-player，支持 HLS/DASH 自适应码率（如果影片多版本）
  • 实现播放控制（播放/暂停、进度条、音量、全屏、下一集）
  • 支持键盘快捷键（空格暂停、左右快进等）
  • 处理移动端浏览器自动播放策略、锁屏控制

B3. 后端与数据库

  • 使用 FastAPI / Spring Boot 构建完整的 REST API
  • 数据库设计：
    • users 表（ID、邮箱、密码哈希、订阅状态）
    • videos 表（标题、描述、分类、封面URL、视频URL、时长、上映年份）
    • watch_history、favorites 等交互表
  • 实现用户注册/登录（JWT 认证）
  • 视频数据 CRUD（后台管理接口）
  • 文件存储：影片文件存至对象存储（OSS/S3），数据库只存 URL，支持防盗链

B4. 推荐与个性化（仿 Netflix 智能）

  • 简单推荐算法：基于影片分类展示“热门”、“新上线”、“因为你看了…”
    （可先基于标签规则实现，后期引入协同过滤）
  • 使用 Redis 缓存首页数据，提高性能
  • 用户观看进度记录，支持“继续观看”功能

B5. 移动端 App 版本

  • 使用 React Native (Expo) 或 Flutter 实现影片浏览与播放 App
  • 登录状态持久化，列表无限滚动，横屏播放
  • 练习原生模块（如播放器、网络状态检测）

B6. AI 智能搜索与问答

  • 为所有影片简介生成 Embedding，存入 Qdrant
  • 后端实现语义搜索接口：输入描述性文字找到对应影片
  • 增加 AI 聊天助手：基于影片库回答用户问题（RAG 模式），比如“推荐一部悬疑短片”

B7. Chrome 插件扩展

  • 开发“随机推荐”插件，点击图标弹出你的影片网站随机一部片

三、双平台运行保障清单

每个项目根目录确保：

  • package.json 脚本使用 cross-env 设置环境变量，无平台特定命令
  • 文件路径统一为正斜杠 / 或使用 path 模块
  • .gitattributes 包含 * text=auto
  • .gitignore 包含 node_modules/, .DS_Store, Thumbs.db, __pycache__
  • 提供 README.md，说明 Windows 和 macOS 下的启动步骤（相同）
  • 若使用 Docker，提供 docker-compose.yml 一键启动 MySQL、Redis、Qdrant
  • 定期将代码克隆到另一系统或虚拟机测试运行

四、技术栈对应技能清单速查

你的目标技能 在哪个项目/阶段涉及
HTML, CSS, JS, TS, ES6 A1, A2, B1, B2
操作系统、计算机网络基础 A3, B3 (部署、流媒体、CORS)
Vue 框架与 UI 库 可根据兴趣替换 React，但建议保持 React 主线
React, Antd, Hooks, Router, Axios A2, B1, B2, B3
H5 移动端开发 A2, B2 移动端适配
Flutter A5 或 B5 可选
React Native A5 或 B5 可选
Cursor/DeepSeek/GPT 等 AI 工具 全部阶段使用
Chrome 插件开发 A5, B7
ECharts 数据可视化 A4
Tailwind CSS A1, B1
Git 协作 全部阶段
Java/Python 后端 A3, B3
MySQL A3, B3
Axure 原型 A1, B1
Webpack/Vite/Less A1, A2 (使用 Vite，了解配置)
Embedding / Qdrant A6, B6

五、立即行动第一步

  1. 游戏作品集网站：

    • 在 Cursor 中输入：

      “帮我用 Vite + React + TypeScript + Tailwind CSS 搭建一个游戏作品集网站，包含游戏卡片网格、详情页和导航，适配移动端，并确保项目能在 Windows 和 Mac 上运行。”

  2. 仿奈飞影片网站：

    • 接着创建另一个项目：

      “帮我用 Vite + React + TypeScript + Tailwind CSS 搭建一个仿 Netflix 风格的视频网站框架，包含横向滚动影片列表、Banner 大图和基本路由，保证跨平台兼容。”

完成骨架后，按清单逐步丰富功能，两个项目交替推进，最终形成一套完整的全栈作品矩阵。
